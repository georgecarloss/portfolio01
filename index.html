Caso Práctico: Easy Loans - Análisis de Préstamos
Introducción
Easy Loans es una empresa financiera que concede préstamos para la adquisición de productos en diferentes establecimientos. Analizaremos las operaciones realizadas durante el ejercicio 2023 utilizando Tableau para extraer insights que apoyen la toma de decisiones estratégicas. Se espera que, a través de este análisis, podamos identificar patrones de comportamiento, evaluar la calidad de los préstamos y proporcionar recomendaciones para mejorar la eficiencia de las operaciones financieras. Además, aplicaremos técnicas avanzadas de visualización y análisis para resaltar las áreas de oportunidad en las operaciones de la empresa.
En este caso práctico desarrollaremos un dashboard de 2 hojas u pestañas en Tableau utilizando visualizaciones y funcionalidad exclusivas de Tableau. Luego trataremos de replicar el dahsboard con PowerBi y Google Looker Studio, herramientas que tienen muchas presencia en el sector de análisis de datos.

Dashboard en Tableau


Como se comentó líneas arriba, optaremos por utilizar algunas visualizaciones y funcionalidades exclusivas del software Tableau, en este caso resaltamos el completo control de los filtros, como usar una barra deslizante en el filtro de fechas en lugar del clasico estilo calendario. También recalcamos el uso de las gráficas de cuadrados para representar de manera visual el agrupamiento de las operaciones. Por último comentar que en Tableau se dispone de un control total de todos los filtos a nivel de informe global permitiendo la óptima sincronización entre hojas de trabajo de manera muy intuitiva.


   

Dashboard adaptado en PowerBi


Ahora pasaremos a adaptar el dahsboard principal en PowerBi, donde recalcamos que de igual forma el desarrollo de informes es muy intuitivo, recalcamos que para esta adaptación el gráfico de areas apiladas y el gráfico de cuadrados en ambas Hojas de trabajo, fueron desarrolladas medainte scripts Python dado que no se disponen de manera nativa en PowerBi, en el caso del gráfico de las áreas apiladas se desarrolló uno propio debido a que la funcionalidad deseada para replicar el dashboard original no se pudo lograr en la visualización nativa.
   

    
        

Dashboard adaptado en Google Looker Studio 2


Para la adaptación en Google Looker Studio, después de muchos tests optamos por reemplazar el gráfico de cuadrados por un gráfico de barras y su Promedio para poder replicar la distinción entre operaciones sobre el promedio y operaciones por debajo del promedio.
